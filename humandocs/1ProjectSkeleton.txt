 ---
 Proto Definitions (proto/events.proto)

 Key messages:
 - MarketEvent — market_id, outcome, probability [0–1], delta, timestamp
 - FightStatEvent — fight_id, fighter_name, stat_type, value, round, timestamp  (all fields defined even though data is stubbed — C++
 compiles clean)
 - CanonicalEvent — event_id (UUID), source enum, ingested_at, oneof {market_event, fight_event}
 - RagPrediction — explanation, evidence[], confidence, timestamp, trigger_event_id
 - EvidenceItem — text, source_ref, score

 gRPC services:
 - EventIngestion (C++ server, Python agents call it) — IngestEvent + IngestStream
 - EventStream (C++ server, gateway calls it) — Subscribe → server-streaming
 - RagStream (Python RAG server, gateway calls it) — SubscribePredictions → server-streaming

 ---
 C++ Engine — Key Design Points

 - EventStore: Fixed-capacity ring buffer, std::shared_mutex for thread safety, cursor-based subscribers so readers never block writers.
 TODO: evaluate boost::lockfree if profiling shows contention.
 - Normalizer: Stateless — validates timestamp within ±60s, probability in [0,1], non-empty IDs. Assigns UUID event_id and ingested_at.
 Thread-safe without locks.
 - GrpcServer: Two service impls sharing EventStore and Normalizer via shared_ptr. EventStreamServiceImpl::Subscribe loops on GetSince,
 uses condition_variable to avoid busy-wait.
 - CMakeLists.txt: C++20, finds gRPC + Protobuf + nlohmann_json + GTest. Compiles proto files, builds engine_lib shared between main
 executable and tests.

 Unit tests (GTest):
 - test_normalizer.cpp — valid/invalid inputs, UUID assignment, timestamp checks
 - test_event_store.cpp — append, cursor, ring overflow, concurrent reads under TSan

 ---
 Python Agents — Key Design Points

 Shared:
 - event_emitter.py — single gRPC channel, reused across calls, exponential backoff on failure
 - retry.py — decorator with backoff + jitter; distinguishes retryable vs non-retryable errors

 Polymarket (Saify):
 - client.py — async aiohttp, caches market IDs (refresh every 5 min), respects 429 headers, saves raw JSON for debugging
 - agent.py — poll loop every 5s, delta threshold 0.01, emits on meaningful change
 - API tests: market list non-empty, prices in [0,1], rate limit respected

 MMA (Saify):
 - client.py — free-tier: get_live_events() active. GOAT-tier methods (get_fight_stats, get_round_stats) raise
 NotImplementedError("BallDontLie GOAT tier required") with clear comment
 - agent.py — discovers live fights, emits FIGHT_DISCOVERED synthetic events; logs that stats polling is disabled
 - API tests: live events list, GOAT-tier raises NotImplementedError

 ---
 RAG Orchestrator — Key Design Points (FWS)

 - orchestrator.py — subscribes to engine EventStream, decides if event is "meaningful", routes to context_builder → retriever → inference
 → verifier → emit RagPrediction. Also implements RagStream gRPC server for gateway to subscribe.
 - retriever.py — Pinecone with two namespaces: market_events + fight_events. Embedding via Gemini text-embedding-004. upsert() on every
 meaningful event (builds store over time for future historical queries). retrieve() returns EvidenceItem[].
 - inference.py — google-generativeai SDK, model gemini-1.5-flash. Prompt: system sets analyst role, user provides event + context +
 evidence + asks for 2-3 sentence explanation + confidence float. Parses confidence from last line.
 - verifier.py — if confidence < 0.5 → neutral message. Basic string containment check that explanation references real
 market_id/fighter_name from triggering event.
 - context_builder.py — deque sliding window of last N events per source; serializes to compact text lines for prompt injection.

 ---
 Gateway — Key Design Points (Samad)

 - server.py — FastAPI, /ws WebSocket endpoint, /health HTTP. On startup (lifespan): starts EngineSubscriber and RagSubscriber as async
 tasks. WS messages are JSON envelopes: {"type": "event", "data": {...}} or {"type": "prediction", "data": {...}}.
 - broadcaster.py — asyncio.Lock-protected set of WebSocket clients. Fan-out with silent removal on send failure. TODO: buffer last 50
 events for late-joining clients.
 - engine_subscriber.py — gRPC client to engine EventStream.Subscribe, converts proto → dict via MessageToDict, pushes to broadcaster with
 type="event".
 - rag_subscriber.py — gRPC client to RAG RagStream.SubscribePredictions, pushes with type="prediction".

 ---
 Frontend — Key Design Points (Zaid)

 Stack: React 18 + Vite + TypeScript + Tailwind CSS

 - useEventStream.ts — manages WS lifecycle, splits messages by type into events[] and predictions[] state, reconnects after 3s on drop,
 trims arrays to max 200/50.
 - Two-column layout: left = EventFeed (Stream 1, factual only), right = PredictionFeed (Stream 2, always shows evidence).
 - EventCard — shows source badge (blue "PM" / orange "MMA"), timestamp, event content. No predictive language.
 - SourceBadge — color-coded pill distinguishing PM vs MMA visually.
 - PredictionCard — explanation + confidence bar (green >0.7, yellow 0.5–0.7) + EvidenceList always visible.
 - StreamWarning — amber banner shown on connection drop or malformed payload.

 ---
 Build Order / Dependencies

 1. proto/ — must exist first; all other components compile/generate from it
 2. engine/ — C++ build (CMake); independently testable
 3. agents/ — depends on generated Python proto stubs
 4. rag/ — depends on generated Python proto stubs + Pinecone + Google AI accounts
 5. gateway/ — depends on engine + rag being reachable (or mocked for unit tests)
 6. dashboard/ — independent; mocks WS with hardcoded JSON for local dev

 Verification

 - C++ engine: cmake --build . && ctest (GTest unit tests + TSan concurrent test)
 - Python agents: pytest agents/tests/unit/ (fast); pytest -m api agents/tests/api/ (live API)
 - RAG: pytest rag/tests/unit/ (mocked); pytest -m api rag/tests/api/ (live Pinecone + Gemini)
 - Gateway: pytest gateway/tests/unit/
 - Frontend: npm test (Vitest); npm run dev + connect to mock WS server to verify both streams render
 - Integration: docker compose up → open browser → observe two-column dashboard populating
